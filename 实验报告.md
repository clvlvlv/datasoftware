### 4.2 归档格式设计

**v3 归档格式（二进制，小端序）：**

```
偏移量  大小  字段
0       6    Magic: "DATASW"
6       2    Reserved: 0x0000
8       4    Version: uint32_t = 3
12      4    EntryCount: uint32_t
16      —    每个条目（重复 EntryCount 次）：
                ┌─ 条目头部 ──────────────────┐
                │ 4 bytes: PathLength         │
                │ N bytes: Path (UTF-8)       │
                │ 1 byte:  FileType           │
                │ N bytes: Type-specific 载荷  │
                │ 8 bytes: CreateTime (int64) │
                │ 8 bytes: ModTime (int64)    │
                │ 8 bytes: AccessTime (int64) │
                │ 4 bytes: Attributes (uint32)│
                │ 2 bytes: OwnerLen (uint16)  │
                │ N bytes: Owner              │
                │ 2 bytes: GroupLen (uint16)  │
                │ N bytes: Group              │
                └─────────────────────────────┘
```

**FileType 枚举：**

| 值 | 类型 | 载荷格式 |
|----|------|---------|
| 0 | Regular | FileSize(8) + FileContent |
| 1 | Symlink | TargetLen(4) + SymlinkTarget |
| 2 | HardLink | FileSize(8) + FileContent + HardLinkId(8) |
| 3 | Fifo | 无 |
| 4 | Device | DeviceMajor(4) + DeviceMinor(4) |
| 5 | Directory | 无 |

### 4.3 加密文件格式

```
偏移量  大小  字段
0       5    Magic: "DSENC"
5       4    Version: uint32_t = 1
9       16   Salt (随机)
25      16   IV (随机初始化向量)
41      N    AES-256-CBC 加密数据
41+N    32   HMAC-SHA256 (salt + IV + 加密数据)
```

### 4.4 压缩文件格式

| 算法 | Magic | 扩展名 | 原理 |
|------|-------|--------|------|
| RLE | DSRL | .rlz | 连续重复字节 → [count, value] |
| LZ77 | DSLZ | .lz77 | 滑动窗口匹配 → [offset, length] |
| Huffman | DSHF | .huf | 变长编码，高频字短码 |


### 7.2 技术难点与解决方案

| 难点 | 解决方案 |
|------|---------|
| Unicode 文件名处理 | 使用 `fs::u8path()` 进行 UTF-8 到路径的转换 |
| 大文件内存溢出 | 4MB 分块流式读写，避免全文件加载 |
| Windows 时间戳纪元差异 | 减去 11644473600 秒偏移（1601→1970） |
| 加密密钥派生 | SHA-256(password + salt) 派生 AES 密钥 |
| HMAC 完整性校验 | 使用 BCrypt HMAC-SHA256 验证数据完整性 |
| 进度条实时更新 | 回调函数 + Qt 信号槽跨线程通信 |

### 7.4 实验心得

通过本次实验，完整经历了一个软件项目从需求分析、系统设计、编码实现到软件测试的全过程。深入理解了：

1. **面向对象设计原则**：通过分层架构实现模块间低耦合，各模块职责清晰
2. **文件系统编程**：掌握了目录遍历、文件元数据读写、符号链接处理等技术
3. **数据压缩算法**：实现了 RLE、LZ77、Huffman 三种经典压缩算法
4. **加密技术**：使用 AES-256-CBC 和 HMAC 保证数据机密性和完整性
5. **GUI 编程**：使用 Qt5 框架开发多标签页图形界面
6. **版本控制**：使用 Git 进行代码管理和协作
